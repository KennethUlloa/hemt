API Documentation

Base URL: {{ api_base_url }}/api/v1

Authentication

All API endpoints require a Bearer token in the Authorization header. Create an API key from the Settings page or via POST /api/v1/api-keys (requires an existing key).

API Keys

API keys authenticate requests and can optionally be assigned a tag. Emails sent with a tagged key are automatically grouped under that tag in your inbox. You can also override the tag per-request with the X-HEMT-Tag header.

GET /api-keys

List all your API keys.

curl http://{{ api_base_url }}/api/v1/api-keys \
  -H "Authorization: Bearer et_abc123..."
{
  "api_keys": [
    {
      "id": 1,
      "name": "my-service",
      "prefix": "a1b2c3d4",
      "scopes": "mail:send,mail:read",
      "tag": "support",
      "is_active": true,
      "created_at": "2026-05-27T12:00:00",
      "last_used_at": "2026-05-27T14:30:00"
    }
  ]
}
POST /api-keys

Create a new named API key with optional scopes and tag.

curl -X POST http://{{ api_base_url }}/api/v1/api-keys \
  -H "Authorization: Bearer et_abc123..." \
  -H "Content-Type: application/json" \
  -d '{"name": "support-bot", "scopes": "mail:send", "tag": "support"}'
{"id": 2, "name": "support-bot", "token": "et_def456...", "scopes": "mail:send", "tag": "support"}
DELETE /api-keys/<id>

Revoke and remove an API key.

curl -X DELETE http://{{ api_base_url }}/api/v1/api-keys/2 \
  -H "Authorization: Bearer et_abc123..."
{"status": "ok"}

Incoming Mail

POST /incoming-mail

Simulate receiving an email. The to address is stored as metadata; no pre-registration required. If the API key has a tag, the message is grouped under that tag. Override per-request with the X-HEMT-Tag header.

Simple text email

curl -X POST http://{{ api_base_url }}/api/v1/incoming-mail \
  -H "Authorization: Bearer et_abc123..." \
  -F "to=inbox@myapp.com" \
  -F "from=user@external.com" \
  -F "subject=Hello" \
  -F "body_text=This is the email body."

With HTML body

curl -X POST http://{{ api_base_url }}/api/v1/incoming-mail \
  -H "Authorization: Bearer et_abc123..." \
  -F "to=inbox@myapp.com" \
  -F "from=user@external.com" \
  -F "subject=Hello" \
  -F 'body_html=<h1>Hello</h1><p>HTML content</p>'

With file attachments

curl -X POST http://{{ api_base_url }}/api/v1/incoming-mail \
  -H "Authorization: Bearer et_abc123..." \
  -F "to=inbox@myapp.com" \
  -F "from=user@external.com" \
  -F "subject=Report" \
  -F "attachment=@/path/to/report.pdf" \
  -F "attachment=@/path/to/image.png"
{"message_id": 42, "status": "ok", "tag": "support"}

Message Retrieval

GET /messages

List messages, newest first. Optionally filter by tag.

curl "http://{{ api_base_url }}/api/v1/messages?page=1&per_page=25" \
  -H "Authorization: Bearer et_abc123..."

curl "http://{{ api_base_url }}/api/v1/messages?tag=support" \
  -H "Authorization: Bearer et_abc123..."
{
  "messages": [
    {
      "id": 1,
      "from_addr": "alice@example.com",
      "to_addr": "inbox@myapp.com",
      "subject": "Hello",
      "body_text": "...",
      "body_html": "",
      "tag": "support",
      "received_at": "2026-05-27T12:00:00"
    }
  ],
  "total": 1,
  "page": 1,
  "per_page": 25,
  "tag": null
}
GET /messages/<id>

Get a single message with its attachments.

curl http://{{ api_base_url }}/api/v1/messages/1 \
  -H "Authorization: Bearer et_abc123..."
{
  "id": 1,
  "from_addr": "alice@example.com",
  "to_addr": "inbox@myapp.com",
  "subject": "Hello",
  "body_text": "...",
  "body_html": "",
  "tag": "support",
  "received_at": "2026-05-27T12:00:00",
  "attachments": [
    {
      "id": 1,
      "filename": "report.pdf",
      "content_type": "application/pdf",
      "size": 10240
    }
  ]
}

Tags

GET /tags

List all tags you've used to organize your messages.

curl http://{{ api_base_url }}/api/v1/tags \
  -H "Authorization: Bearer et_abc123..."
{"tags": ["support", "newsletter", "alerts"]}
